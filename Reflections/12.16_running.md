1.这个测试到底在验证什么语义？

核心函数：NumberUtils.createNumber(String)
Input：形如 "0x80000000"、"0xFFFFFFFFFFFFFFFF" 的 hex 字符串（还包含leading zeros）
期望：根据数值大小，返回最小能容纳该值的数值类型：

小到一定程度 → Integer

超出 int 范围但在 long 范围 → Long

超出 long 范围（或会被解释成负数的边界）→ BigInteger

测试里非常关键的边界点：

0x7FFFFFFF（int 最大值）→ Integer

0x80000000（刚超出 int）→ Long

0xFFFFFFFF（超过 int，但在 long）→ Long

0x7FFFFFFFFFFFFFFF（long 最大值）→ Long

0x8000000000000000（= 2^63，long 无法表示为正数，Java long 会变负）→ BigInteger

0xFFFFFFFFFFFFFFFF（= 2^64-1，long 更不可能）→ BigInteger

并且：前导 0 不应该改变类型选择（leading zero tests）。
